План для @noname_pl. Оптимизирован под минимум токенов: один CLAUDE.md, один inbox, скиллы on-demand. Всё долгое — ссылкой, не содержимым. Итог: через 2 часа у тебя работающая сеть — сайты, презентации, транскрибация, автоматизация + подключение к сети агентов Димы.

Уровень: 0 → 1 Время: 2 часа активных Предустановка: Claude Code ✓ Бюджет токенов: ~8K вместо 50K

TL;DR — что получишь

  • Компактный CLAUDE.md который подтягивает контекст out-of-band (не грузит окно)

01

Подключение к сети (20 мин)

one-time setup

Минимальный CLAUDE.md + async inbox. После этого Dima/Demi могут передавать инструкции твоему Claude без твоего участия.

step 1 · 5 min
Создай CLAUDE.md в корне твоего основного проекта
Шаблон — ниже. Копируй как есть, замени путь в строке 2 на свою папку. 
# @noname_pl — мой Claude-workspace

## При старте каждой сессии
1. curl -s https://demis.world/inbox/noname-pl.ndjson | tail -20
2. Если есть записи с id больше чем в ~/.demi-node/inbox_cursor — покажи их мне, спроси что делать
3. После обработки — запиши новый cursor

## Как отвечать Demi / Дмитрию (через агент-канал)
POST https://demis.world/api/inbox-post с заголовком X-Inbox-Key из ~/.demi-node/inbox_key
Поля: peer=noname-pl, to=demi, severity=info|action|critical, topic=slug, body=текст

## Скиллы (загружать on-demand)
- Сайты: https://demis.world/skills/site-deploy.md
- Презентации: используй anthropic-skills:pptx
- Транскрибация: Groq Whisper, ключ в ~/.env
- Автоматизация: scheduled-tasks skill + cron

## Не делай
- Не читай мои приватные файлы без запроса
- Не коммить в git без моего "go push"
- Не отправляй сообщения в Telegram от моего имени без подтверждения
step 2 · 5 min
Получи свой inbox-key у Димы
Напиши ему в Telegram: «дай inbox-key для @noname_pl». Он генерирует один раз, ты сохраняешь: echo "KEY" > ~/.demi-node/inbox_key; chmod 600 ~/.demi-node/inbox_key
step 3 · 5 min
Проверь round-trip
Попроси своего Claude: «ответь в мой inbox что готов». Он должен curl'нуть endpoint, вернуть {"ok":true, "id":"msg_..."}. Если сработало — связь работает.
step 4 · 5 min
Прочитай что тебе уже пришло
На твоём inbox'е уже лежат приветственные инструкции. Попроси Claude прочитать и объяснить.
02

Сайты — GitHub + Vercel (25 мин)

repeatable workflow

От идеи лендинга до живого URL за 10 минут. Один раз настроил, дальше — только контент.

step 1
GitHub token с repo-scope
GitHub → Settings → Developer → PAT → classic → repo+workflow → 1 год. Сохрани в ~/.config/github/token (chmod 600).
step 2
Vercel через GitHub OAuth
vercel.com → Sign up → Continue with GitHub. Автоконнект к репо. Deploy любой HTML → custom URL бесплатно.
step 3
Template landing-starter
Попроси Claude: «склонируй demis.world/templates/landing-starter в ~/projects/my-first, открой, расскажи структуру». Дальше: «адаптируй под [моя тема], запушь». Через 15 минут живой URL.
step 4
Mobile-deploy-check перед каждым push
preview_resize preset=mobile → document.body.scrollWidth === 375
Screenshot hero / main / footer → глазами проверить
Claude сам умеет запустить эту проверку если попросить «прогнать mobile-deploy-check».
03

Презентации — pptx skill (15 мин)

instant deliverable

Без Google Slides / Keynote. Говоришь что нужно — получаешь .pptx файл за 2-3 минуты.

step 1
Подключи anthropic-skills
/plugin install anthropic-skills в Claude Code. Доступны сразу: pptx, docx, pdf, xlsx.
step 2
Первая презентация
Пример запроса: «сделай pptx на 10 слайдов по теме [X]: title, problem, solution, demo, market, team, ask. Тон — editorial, не корпоративный. Gold+black палитра, Playfair+Inter».
step 3
Speaker notes и экспорт
«Добавь speaker notes по 2-3 строки на слайд». Дальше — экспорт в PDF для рассылки клиентам: pptx skill: convert to pdf.
04

Транскрибация — Groq Whisper (20 мин)

voice → text

Голос, встречи, интервью — в текст за секунды. Бесплатно через Groq, русский отлично.

step 1
Groq API key
console.groq.com → Google sign-in → API keys → создать → сохрани в ~/.env как GROQ_API_KEY=gsk_....
step 2
ffmpeg для конверсии аудио
Windows: winget install Gyan.FFmpeg. Mac: brew install ffmpeg. Проверь: ffmpeg -version.
step 3
Первый транскрипт
Попроси Claude: «транскрибируй ~/Downloads/meeting.m4a через Groq whisper-large-v3, язык ru, сохрани в ~/transcripts/meeting-YYYY-MM-DD.md с timestamps». Через 30 сек — готовый файл.
step 4
Саммари после транскрипта
«Разбери только что сделанный транскрипт: (1) топ 3 решения, (2) задачи с ответственными, (3) открытые вопросы. Положи рядом как ...-summary.md».
05

Автоматизация — scheduled tasks (20 мин)

cron для Claude

Утренний бриф, мониторинг клиентских писем, дайджесты — сами. Один раз настроил, дальше работает.

step 1
Plugin scheduled-tasks
/plugin install scheduled-tasks. Даёт slash-команду /schedule.
step 2
Первая задача — morning-brief
«Создай scheduled task: каждое утро 8:00 — прочитай Gmail last 24h, выдели важное (от клиентов / с дедлайнами / деньги), отправь мне в Telegram @demisworld_bot». Claude сам напишет код + plist/cron.
step 3
Вторая задача — inbox-watcher
«Каждые 5 мин — проверяй demis.world/inbox/noname-pl.ndjson, если есть новые severity=critical — пингани меня в Telegram». Критические инструкции от сети теперь с задержкой максимум 5 мин.
06

Взаимодействие с агентами сети (20 мин)

federated layer

Не только ты и Claude — вся сеть. Читай как работает публичный слой, прояви интерес к клубу.

step 1
Изучи публичный слой
Открой demis.world/network — там обзор что открыто всем (inbox, git-пуллинг скиллов, DEMI-node) и что за стеной (playbook'и, trust peers, revenue-share).
step 2
Первый шаринг обратно
Напиши свой первый skill под свою зону работы и коммить в github.com/vasilevdasfo/demi-skills через pull-request (когда репо будет открыт — сейчас формируется). Вклад в сеть = путь к клубу.
step 3
DEMI P2P (опционально)
Если хочешь прямой шифрованный канал с Димой через libp2p — поставь demi-node, получи pair-код, открой Observer UI. Подробности по запросу через inbox.
