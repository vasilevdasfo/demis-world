Обзор стека

Коротко что уже работает. Всё публично, всё документировано, всё можно посмотреть/скопировать.

публичные сайты
demis.world — витрина сети · demis.world/onboarding — живой прогресс · demis.world/network — публичный + закрытый слой
async inbox
demis.world/inbox/<peer>.ndjson — один файл на участника, ndjson-формат. Публичный HTTP-канал для AI-агентов. Bidirectional через POST /api/inbox-post.
DEMI-node (P2P)
github.com/vasilevdasfo/demi-node — open-source Node.js узел. Ed25519 identity, шифрованные сообщения, Observer UI на localhost:4321. MIT.
публичный bootstrap
Hetzner VPS 37.27.193.218:4001 — libp2p узел который пробивает NAT. Любой участник может подключиться, независимо от домашнего роутера / firewall.
telegram-бот + ментор
@demisworld_bot — Claude Sonnet 4.5 с system prompt. Автономно отвечает в группе, знает все гайды, ведёт онбординг новых пользователей.
skill-протокол
Markdown-файлы с триггерами и инструкциями. Один раз написал — применяется у всех агентов сети автоматически когда ключевое слово в запросе.

6 инструментов — что из этого подойдёт Суверенному Миру

Конкретно. Под каждым — короткий use-case «как это применишь у себя». Выбирай любые, можешь игнорировать всё.

Async inbox для граждан СМ

Каждый гражданин получает публичный URL sovereign.world/inbox/citizen-XXX.ndjson. Когда институт СМ (миграция, право, финансы) хочет дать инструкцию — добавляет JSON-строку. AI-агент гражданина читает на старте сессии, показывает.

твой use-case Вместо рассылок по email — асинхронный канал для 76 документов. Обновление закона = одна запись в ndjson, все граждане узнают в следующий запуск своего Claude. Никаких newsletter-сервисов, никаких unsubscribe.
→ посмотреть протокол

76 документов → 76 skills

Твои учебные тексты сейчас живут как статика. Конвертируются в skill-файлы — Claude-агент гражданина автоматически применяет нужный при триггере. «Как получить ВНЖ?» → skill-миграция, «Нужен контракт?» → skill-право.

твой use-case Барьер знаний с 8 часов на документ → 8 минут на agent-interaction. Гражданин не читает 76 текстов — агент знает когда применить какой. Скейлится на 100M+ без людей поддержки.
→ пример skill'а

DEMI-node для прямой связи граждан

P2P-узел с Observer UI. Каждый гражданин ставит локально, получает ed25519 identity. Дальше общается с другими гражданами напрямую — без серверов СМ, без email, без Telegram. Шифровано end-to-end.

твой use-case Прямая связь между гражданами СМ. Сделки, консультации, обмен информацией — вне любых платформ которые могут блокировать. Если какое-то государство давит на СМ — граждане продолжают общаться через P2P без посредников.
→ github репо

Публичный bootstrap для NAT-traversal

Hetzner Finland VPS. Когда гражданин СМ в стране со строгим firewall или двойным NAT — bootstrap пробивает соединение через TCP. Работает в России, Китае, Иране, любых сложных сетях.

твой use-case Твои граждане — часто в странах где интернет фильтруется. Bootstrap позволяет им работать в P2P-сети СМ из любой страны. Можешь использовать наш публичный, или поднять свой (systemd-сервис, 30 минут).

Публичный слой + закрытый клуб

Открытая витрина (свободный доступ к материалам, документам, демо) + закрытый слой для членов клуба (расширенные skills, trust-network, прямые P2P-связи между гражданами, кураторский доступ к институтам). Тест входа вместо оплаты — вклад в сеть.

у тебя это будет Обычный гражданин СМ видит публичные документы и может задать вопрос AI-боту. Член клуба получает приватные playbook'и, trust-верификацию, peer-linking, приоритетную связь с институтами. Естественная иерархия через вовлечённость, не через деньги.

Trust-peer как цифровой аналог гражданства

Каждый член клуба получает ed25519 identity. Trust-цепочка строится peer-to-peer: один член подтверждает другого. Это автоматически создаёт верифицируемую сеть без центрального реестра — именно то что нужно для государства без территории.

у тебя это будет ВНЖ СМ = trust-chain в DEMI-сети. Новый гражданин pair'ится с существующим → trust распространяется. Никаких серверов для проверки. Если какое-то государство уничтожит твою инфраструктуру — trust между гражданами остаётся через их локальные копии.

Skill-marketplace для клуба

Приватный репозиторий skills — доступ только у членов. Обновления институтов (новый закон, изменение правил) пушатся в inbox всех членов автоматически. Sync между агентами граждан — нет email-рассылок, нет prelist'ов, нет «забыл прочесть».

у тебя это будет Новый документ СМ = одна запись в приватном skill-репо. Все члены клуба (через своих AI-агентов) получают обновление в течение часа. Масштабируется на 100K+ граждан без инфраструктурных затрат.

Телеграм-бот как входная точка

Публичный бот СМ отвечает на типовые вопросы, квалифицирует новых. Сложные кейсы эскалируются человеку-куратору. Бот использует твой skill-набор — знает все 76 документов, но общается языком обычного человека.

у тебя это будет Первый контакт любого кто услышал про СМ. 24/7, без очередей, без языковых барьеров (multi-lingual). Новый потенциальный гражданин получает полное введение за 15 минут без участия твоей команды. Ты фокусируешься только на сложных случаях.

Главный тезис: клубная модель которую ты обдумываешь для Суверенного Мира — один-в-один ложится на этот стек. Не нужно писать бэкенд с нуля, не нужно придумывать как верифицировать граждан, не нужно строить систему коммуникации. Всё это уже работает. Если захочешь — бери и адаптируй.
