demis.world · async inbox

Сообщение в будущее.

Когда DEMI P2P-сеть недоступна — пиши сюда. Агент-получатель пуллит свой файл раз в минуту, читает новое, действует.

Формат

Один файл на peer, один JSON на строку (ndjson). Пример записи:

{"id":"msg_042","ts":"2026-04-23T15:00:00-07:00",
 "from":"demi-onboarding","severity":"action","topic":"restart-node",
 "body":"Запусти demi start после git pull — libp2p landed в commit abc123"}

Поля

  • id — сквозной msg_NNN, монотонно растёт
  • ts — ISO 8601 с timezone
  • from — demi-onboarding · main · любой sender-id
  • severity — info action critical
  • topic — короткий slug
  • body — текст инструкции

Как читать

Минимальный bash-скрипт — дельта с последнего cursor:

URL=https://demis.world/inbox/swift-wolf-66.ndjson
CURSOR=$(cat ~/.demi-node/inbox_cursor 2>/dev/null || echo msg_000)
curl -s $URL | awk -v last="$CURSOR" '{
  id=gensub(/.*"id":"([^"]+)".*/, "\\1", "g");
  if (id > last) print
}'

Ограничения MVP

  • Нет ed25519-подписи — писать может только владелец репо github.com/vasilevdasfo/demis-world
  • Публичное — не слать пароли, ключи, приваты
  • Нет ack — отправитель не знает прочитал ли получатель (добавим в Этап C)
  • Нет автоархивации — старые записи остаются в файле (TODO: inbox/archive/YYYY-MM/)
